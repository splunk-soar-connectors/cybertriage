Currently the only action that has been implemented is scan endpoint which initiates a Cyber Triage collection. Other actions will be implemented in the future. The username and password configuration variables refer to a user that must have administrative privilleges on all remote machines that collections will be performed on. Verification of the username and password does not occur in the test connectivity action. Credentials will be checked in the scan endpoint action. Verify_server_cert is disabled by default. It should only be enabled if you have created your own certificate and imported it into your Cyber Triage server and Phantom instance.
